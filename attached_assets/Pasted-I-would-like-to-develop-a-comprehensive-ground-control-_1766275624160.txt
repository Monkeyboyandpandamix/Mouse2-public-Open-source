I would like to develop a comprehensive ground control station application similar to Mission Planner, optimized to run on a Raspberry Pi for real-time autonomous drone control. The system will feature a Raspberry Pi as the companion computer connected to an Orange Cube+ flight controller, with the Orange Cube's I2C Port 2 connected to a LiDAR sensor for precision distance measurement and obstacle detection, and its CAN 2 port connected to a HERE3+ GPS/compass module for navigation. A CAN 2 bus splitter will provide real-time telemetry from the drone's motors and ESCs, while a USB-connected buzzer will handle audio alerts. The Raspberry Pi will also interface with a dual-camera gimbal system featuring both standard RGB and thermal imaging cameras for enhanced situational awareness.

The drone must be capable of operating in GPS-denied environments using visual odometry, LiDAR distance, and inertial navigation as fallback systems. The application needs to incorporate real-time computer vision for object detection and classification, with the ability to automatically track and follow moving targets while making intelligent obstacle avoidance decisions by fusing data from both the LiDAR sensor and camera feeds. Additionally, the system should generate 3D reconstructions of the environment, mapping buildings, terrain, and objects from the camera input in real-time.
The architecture will be distributed across two installations of the application: one running on the Raspberry Pi as the onboard computer and another on a desktop or laptop serving as the primary ground control station. Both systems will communicate via WiFi, with the ground station sending mission commands to the Raspberry Pi, which then translates these into flight controller instructions for the Orange Cube+ while processing all sensor data streams and executing computer vision algorithms locally. The Raspberry Pi must continuously buffer telemetry data and record flight footage to handle temporary communication dropouts, automatically synchronizing when connectivity is restored.
The user interface needs to provide multiple map viewing options including satellite imagery, 2D street maps, and hybrid views, all displaying the drone's real-time position and flight path history. Users should be able to search and enter street addresses as waypoints and drop down pin, with the system automatically planning and executing routes to those locations.
